3-Angle Axon-Severing Microfluidic Device

PDMSモールド設計書 — v5 | build_tri_angles.py | 2026-03-08

1. Why — なぜこのデバイスを作るのか

目的

ニューロンの軸索を選択的に物理切断し、切断後の変性・再生を観察するためのPDMSマイクロ流体デバイス。

原理

  • 3つのウェル (W1, W2, W3) を三角形に配置し、ウェル間を75μm幅の軸索チャネルで接続
  • 各軸索チャネルに直交する針挿入流路（200μm針用）で、選択的に片方の経路だけ切断可能
  • PDMSモールド（凸形状）→ PDMS鋳造（溝/穴）→ ガラス基板に貼り付け → 細胞培養

なぜ 70°/90°/110° の3角度か

Withers et al. (2006) により、軸索の成長円錐は90°を境に旋回行動が大きく変わることが確立されている （45°: 旋回率50-60%, 90°: 60-85%, 135°: 75-95%）。 90°を中心に ±20° で対称サンプリングすることで：

角度軸索の振る舞い切断実験での意義
70° (鋭角)Path2 に入りやすい両方の path に軸索が伸びる → 同時切断リスクの検証
90° (直角)臨界角度（基準）標準的な分岐条件
110° (鈍角)Path2 に入りにくい選択的切断に最適な角度か検証

3角度 × 2 = 6デバイスを1トレイに配置し、1回の実験で統計比較を可能にする。

2c. 断面図 (高さ関係)

Well (500μm) がリム (400μm) より 100μm 高い → PDMS を rim まで流し込むと well 頂部が露出 → 細胞播種口になる。 全フィーチャーは 50μm 土台に埋め込み (embed)。

Step 0: /delete_everything — キャンバスクリア

Step 1: ベースプレート
  /Box — 計算された bp_wx × bp_hz × (BASE_THICK + EMBED)
  z_off = -BASE_THICK (土台は床面より下に沈む)

  上下リム: X方向に /Box を配置。カラム境界に AIR_GAP_TB のギャップ

Step 3: デバイス ×6 (各 angle で以下を繰り返す)
  θ = angle, sin_t = sin(θ), cos_t = cos(θ)
  W3 = (-D·sinθ, BRANCH_Z + D·cosθ)
  Path2方向: p2 = (-sinθ, cosθ)
  Needle2方向: n2 = (-cosθ, -sinθ)
  Needle2中心: midpoint of (0, BRANCH_Z) and W3

  3a. Wells ×3: /draw_cylinder (W1, W2, W3)
  3b. Needle1 (水平):
      Seal1:  /Box at (0, N1_Z)
      Guide1-ins: /Box at (0.125, N1_Z)
      Guide1-noins: /Box at (-0.09, N1_Z)
      Taper1: /draw_lines (台形4点) → /extrude_last_sketch
  3c. Needle2 (回転): rotated_rect/trap で4頂点を計算

  3d. Path1a: /Box (W1下端 → Seal1上端)

  3e. Path2a: /draw_lines (分岐点 → Seal2手前) → /extrude

Step 4: /join_all_bodies — バッチ結合 (20ボディずつ)

3c. 座標計算の要点

ベースプレートの座標:
bp_x_min = dev_x_min - RIM_PAD - RIM_W
bp_x_max = dev_x_max + (N_COLS-1)·x_step + RIM_PAD + RIM_W
bp_z_min = dev_z_min - RIM_PAD - RIM_W
bp_z_max = dev_z_max + (N_ROWS-1)·z_step + RIM_PAD + RIM_W
ここで x_step = x_span + GAP, z_step = z_span + GAP, dev_x/z_min/max は全角度の bounding box の union。
回転フィーチャーの頂点計算:
Needle2 の Seal/Guide/Taper は rotated_rect(cx, cz, dx, dz, px, pz, hw, start, end) で計算。
(dx,dz) = 長さ方向 (n2), (px,pz) = 幅方向 (p2), hw = 半幅, start/end = 中心からの距離。
台形は rotated_trap(..., hw_start, hw_end, ...) で start/end で異なる半幅。
デバイス配置オフセット:
xo = col · x_step, syo = row · z_step
全フィーチャー座標に (xo, syo) を加算。リムは bp_* 座標で独立配置。
